# role_plugin

## 描述
角色 — 按需聘请的专家

## 核心概念：按需加载，用完即释放

角色是"按需聘请的专家"：
- **切换角色时**：清空 L1（conversation_history），加载新角色记忆到 System Prompt
- **离开角色时**：角色记忆自动从 L1 消失
- **本能信息仍然持续注入**（与角色无关的教训、成功经验）

## 工具

| 工具 | 参数 | 描述 |
|------|------|------|
| list_roles | 无 | 列出所有可用角色 |
| get_role_info | role_name | 查看角色详情 |
| create_role | role_name, role_md, plugins_json, memory_md | 创造新角色 |
| switch_role | role_name | 切换角色（自动清空 L1 + 加载新角色记忆） |

## 切换角色流程

1. 持久化当前角色记忆到 Disk（如果有工作笔记）
2. **不清空 L1** — 所有角色共享同一个对话历史
3. 加载新角色身份到 System Prompt
4. 插件由角色**自己按需加载**（核心插件预加载，非核心插件槽位用完就空着）
5. 本能信息仍然持续注入

**多角色接力**：多个角色可以接力处理同一个任务，每个角色都能看到完整的对话历史。

**非核心插件槽位**：
- 同一时间只容纳一个非核心插件
- 加载新非核心插件时自动卸载旧的
- 切换角色时，如果新角色不需要非核心插件，槽位自动清空

## 与本能的区别

| 组件 | 是否随角色切换 |
|------|---------------|
| L1 (对话历史) | **共享**，不清空 |
| 核心插件 | 预加载，始终可用 |
| 非核心插件槽位 | 用完即空，切换时自动清理 |
| 角色记忆 (memory.md) | 切换时加载/卸载 |
| 本能信息 (lessons/wins) | 持续注入，不受角色影响 |

## 决策指南

- **需要专业视角** → 切换到对应角色（角色记忆被加载）
- **不需要角色了** → 切换到无角色或切换到其他角色（自动释放）
- **本能信息** → 始终存在，不需要角色切换即可访问

## 创建角色时机

遇到以下情况时，可以考虑**创建新角色**：

| 时机 | 示例 | 创建理由 |
|------|------|----------|
| **反复解决同类问题** | 多次写爬虫代码 | 把经验固化为 coder 角色 |
| **需要特定记忆** | 用户偏好某编程风格 | 创建角色存储偏好 |
| **组合多种能力** | 编程+写作+分析 | 创建综合角色 |
| **现有角色不够用** | 没有能解决问题的角色 | 按需创建 |

**创建角色的判断流程**：
```
遇到问题
  ↓
list_roles() 查看现有角色
  ↓
现有角色无法高效解决？
  ├─ 是 → create_role() 创建新角色
  └─ 否 → 切换到合适的现有角色
```

## 创建角色最佳实践

```
create_role(
    role_name="xxx",        # 简洁有力的名字
    role_md="xxx",          # 一句话描述身份和职责
    plugins_json="[]",       # 该角色需要的插件列表（可选）
    memory_md=""             # 初始记忆（可选）
)
```

**原则**：
- 角色名要**有意义**，AI 能推断用途
- role_md 要**简洁**，AI 能快速理解角色定位
- 角色解决**一类问题**，不要一个角色解决所有问题

## 本能注入（自动）

| 本能 | 触发条件 | 注入内容 |
|------|----------|----------|
| role_guide | 始终 | 角色切换指南（何时切换、怎么用） |
| role_creation_guide | 始终 | 创建角色时机和方法论 |

## 角色层级

- **主角色（永恒）**：名字、性格、与用户的情感连接，是你的根基
- **专家角色（临时）**：工作方法论、专业知识，完成任务后可切回

**重要**：主角色永远存在，专家角色只是叠加在工作上
