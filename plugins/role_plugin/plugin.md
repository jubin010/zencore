# role_plugin

## 描述
角色 — 角色 = 身份 + 记忆 + 插件清单

## 触发词
角色、身份、切换、扮演、人格

## 工具
| 工具 | 参数 | 描述 |
|------|------|------|
| list_roles | 无 | 列出所有可用角色 |
| get_role_info | role_name: str | 查看角色详情 |
| create_role | role_name, role_md, plugins_json, memory_md | 创造新角色 |
| switch_role | role_name: str | 切换角色身份和记忆 |

## 🧠 角色方法论（必读）

### 1. 角色与插件是正交的
* **角色（Role）= 灵魂**（身份 + 记忆）：解决"我是谁？我站在什么角度思考？"
* **插件（Plugin）= 工具**（能力）：解决"我能做什么？"
* **同一个工具可以被不同角色使用**。

### 2. 圆桌会议模式
* 所有角色共享同一个 `conversation_history`（对话历史）。
* 这就像一个**圆桌会议**：换一个人站起来发言，但所有人都能看见白板上的内容。

### 3. 如何使用
1. 用 `list_roles` 查看可用角色。
2. 用 `get_role_info` 了解角色的身份、所需插件清单和记忆。
3. 用 `switch_role` 切换身份和记忆。
4. **自主决定**是否需要 `load_plugin` 加载角色建议的插件。
5. 如果现有角色都不合适，用 `create_role` 创造全新角色。

### 4. 决策指南
* **用户需求变化大** → 切换角色（换脑子）。
* **需要新能力** → 加载插件（换工具）。
* **两者都需要** → 先切换角色，再按需加载插件。
