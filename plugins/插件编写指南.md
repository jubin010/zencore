# 插件编写指南

> 一切皆为插件 — AgentCore 的所有能力都由插件提供

## 渐进式披露规范

**核心原则**：API Schema 中的描述必须极简（4-16 字符），详细说明写在 `plugin.md` 中。

| 层级 | 内容 | 描述长度 | 获取方式 |
|------|------|---------|---------|
| **API Schema** | 工具名 + 极简描述 + 参数 | 4-16 字符 | 系统自动传入 LLM |
| **Plugin Index** | 插件列表 | 一句话 | `plugins.md` 注入 System Prompt |
| **Plugin Manual** | 详细用法、示例、注意事项 | 完整 | AI 按需读取 `plugin.md` |

**错误示例**（描述太长）：
```python
"description": "覆盖写入全局记忆。用于更新用户画像、项目全局配置等。"  # ❌ 太长
```

**正确示例**（极简描述）：
```python
"description": "覆盖写入全局记忆"  # ✅ 8 字符
```

详细说明、使用场景、注意事项 → 写在 `plugin.md` 中。

## 目录结构

每个插件是一个独立目录：

```
plugins/
├── my_plugin/
│   ├── __init__.py    # 插件代码（必须有 register 函数）
│   └── plugin.md      # 工具说明文档（强烈建议）
└── plugins.md         # 插件索引（自动生成）
```

## __init__.py 规范

每个插件必须包含一个 `register(agent)` 函数：

```python
"""
my_plugin - 一句话描述插件功能
"""

from pathlib import Path

PLUGINS_DIR = Path(__file__).parent.parent


def register(agent):
    """注册插件到 AgentCore"""
    
    def my_tool(param: str) -> str:
        """工具函数"""
        return f"结果: {param}"
    
    # 注册工具（必须包含 plugin 字段）
    # description 必须极简（4-16 字符），只说明"做什么"
    # 详细用法、示例、注意事项写在 plugin.md 中
    agent.add_tool("my_tool", my_tool, {
        "name": "my_tool",
        "description": "工具做什么",       # ← 极简！
        "parameters": {
            "type": "object",
            "properties": {
                "param": {
                    "type": "string",
                    "description": "参数描述"
                }
            },
            "required": ["param"]
        },
        "plugin": "my_plugin"    # ← 必须与目录名一致
    })
    
    # 返回插件信息
    return {
        "name": "my_plugin",
        "version": "1.0.0",
        "author": "作者名",
        "description": "一句话描述插件",
        "tools": ["my_tool"]
    }
```

## 关键规则

1. **必须有 `register(agent)` 函数** — 这是插件入口
2. **每个工具必须带 `"plugin": "目录名"`** — 用于热载入时清理旧工具
3. **工具函数返回值必须是字符串** — AI 只能理解文本
4. **工具函数必须处理异常** — 返回错误信息而不是抛异常
5. **`__init__.py` 尽量简洁** — 减少模块缓存问题
6. **description 必须极简（4-16 字符）** — 详细说明写进 `plugin.md`
7. **每个插件必须有 `plugin.md`** — 这是渐进式披露的第三层

## plugin.md 规范

`plugin.md` 是工具的详细说明文档，包含 API Schema 中不写的细节：

```markdown
# my_plugin

一句话描述插件功能

## 工具

### my_tool
工具做什么。
- `param`（必填）：参数描述
- 返回：返回值说明

## 使用示例

```
my_tool(param="test") → "结果: test"
```

## 注意事项
- 依赖条件（如需要网络、需要 API key 等）
- 已知限制
- 特殊说明
```

### plugin.md 编写要点

- **工具部分**：逐个工具说明，包括参数含义、返回值、边界情况
- **使用示例**：给出 1-2 个典型调用示例
- **注意事项**：依赖、限制、与其他工具的配合方式
- **不要重复 API Schema 中已有的内容** — 只补充细节

## 编写流程

### AI 自动编写
```
1. 判断需要什么功能
2. get_plugin_template("新插件名") 获取模板
3. 修改模板代码，遵循渐进式披露规范
4. write_plugin("新插件名", code, plugin_md)
5. 自动加载并更新索引
```

### 人类手动编写
```bash
mkdir plugins/my_plugin
# 编写 __init__.py 和 plugin.md
# 调用 reload_plugins() 或重启生效
```

## 多工具插件示例

```python
def register(agent):
    def tool_a(x: str) -> str:
        return f"A: {x}"
    
    def tool_b(y: int) -> str:
        return f"B: {y * 2}"
    
    agent.add_tool("tool_a", tool_a, {
        "name": "tool_a",
        "description": "工具A做什么",     # ← 极简
        "parameters": {
            "type": "object",
            "properties": {
                "x": {"type": "string", "description": "输入字符串"}
            },
            "required": ["x"]
        },
        "plugin": "my_plugin"
    })
    
    agent.add_tool("tool_b", tool_b, {
        "name": "tool_b",
        "description": "工具B做什么",     # ← 极简
        "parameters": {
            "type": "object",
            "properties": {
                "y": {"type": "integer", "description": "输入整数"}
            },
            "required": ["y"]
        },
        "plugin": "my_plugin"
    })
    
    return {
        "name": "my_plugin",
        "version": "1.0.0",
        "author": "作者",
        "description": "多工具插件",
        "tools": ["tool_a", "tool_b"]
    }
```

对应的 `plugin.md`：

```markdown
# my_plugin

提供工具A和工具B两个功能。

## 工具

### tool_a
处理字符串输入。
- `x`（必填）：输入字符串
- 返回：`"A: {x}"`

### tool_b
处理整数输入并翻倍。
- `y`（必填）：输入整数
- 返回：`"B: {y * 2}"`

## 使用示例

```
tool_a(x="hello") → "A: hello"
tool_b(y=21) → "B: 42"
```

## 注意事项
- 两个工具互不依赖，可独立调用
- tool_b 只接受整数，传入字符串会报错
```
